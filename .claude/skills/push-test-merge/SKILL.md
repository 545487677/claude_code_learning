# Push → Test → Merge 自动化流程

你是一个GitHub自动化助手。当用户调用 `/push-test-merge` 时，执行以下完整流程。

## 执行步骤

### 步骤1: 检查当前状态
```bash
# 检查Git状态
git status
git branch
git log --oneline -3
```

向用户展示当前状态并询问：
- 是否确认要推送这些更改？
- 目标分支是什么？（默认：当前分支）

### 步骤2: 确保SSH配置
```bash
# 检查远程仓库地址
git remote -v

# 如果是HTTPS，切换到SSH
# git remote set-url origin git@github.com:USER/REPO.git
```

如果是HTTPS地址，自动切换到SSH协议。

### 步骤3: 推送代码
```bash
# 如果有未提交的更改，先提交
git add .
git commit -m "自动提交：由 push-test-merge skill 创建"

# 推送到远程
git push origin <current-branch>
```

### 步骤4: 运行测试（如果存在）
自动检测并运行项目测试：

**Node.js项目**:
```bash
if [ -f "package.json" ]; then
  npm test || yarn test
fi
```

**Python项目**:
```bash
if [ -f "pytest.ini" ] || [ -f "setup.py" ]; then
  pytest
fi
```

**Go项目**:
```bash
if [ -f "go.mod" ]; then
  go test ./...
fi
```

**自定义测试脚本**:
```bash
if [ -f "scripts/test.sh" ]; then
  ./scripts/test.sh
fi
```

如果测试失败，停止流程并报告错误。

### 步骤5: 创建PR（如果不在main分支）
如果当前分支不是 `main` 或 `master`，使用GitHub API创建PR：

```bash
# 获取当前分支和仓库信息
CURRENT_BRANCH=$(git branch --show-current)
REPO_INFO=$(git remote get-url origin | sed 's/.*github.com[:/]\(.*\)\.git/\1/')

# 创建PR
curl -X POST \
  -H "Authorization: token ${GITHUB_PERSONAL_ACCESS_TOKEN}" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/${REPO_INFO}/pulls \
  -d '{
    "title": "Auto PR: '"${CURRENT_BRANCH}"'",
    "head": "'"${CURRENT_BRANCH}"'",
    "base": "main",
    "body": "## 自动创建的PR\n\n由 push-test-merge skill 自动生成\n\n### 变更内容\n[描述变更]\n\n### 测试状态\n✅ 所有测试通过\n\n🤖 Generated by Claude Code"
  }'
```

### 步骤6: 合并PR（可选）
询问用户是否立即合并PR。如果确认：

```bash
# 合并PR（使用squash方式）
curl -X PUT \
  -H "Authorization: token ${GITHUB_PERSONAL_ACCESS_TOKEN}" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/${REPO_INFO}/pulls/${PR_NUMBER}/merge \
  -d '{
    "merge_method": "squash",
    "commit_title": "Merge: '"${CURRENT_BRANCH}"'",
    "commit_message": "自动合并 by push-test-merge skill"
  }'
```

### 步骤7: 清理分支
合并后清理所有分支，只保留main分支：

```bash
# 切换回main
git checkout main
git pull origin main

# 删除当前工作分支
git branch -D ${CURRENT_BRANCH}
git push origin --delete ${CURRENT_BRANCH}

# 清理所有其他分支（只保留main）
echo "🧹 清理其他分支..."

# 删除所有本地分支（除main外）
LOCAL_BRANCHES=$(git branch | grep -v "main" | grep -v "\*" | xargs)
if [ -n "$LOCAL_BRANCHES" ]; then
  for branch in $LOCAL_BRANCHES; do
    git branch -D "$branch" && echo "✅ 删除本地分支: $branch"
  done
else
  echo "ℹ️  没有其他本地分支需要清理"
fi

# 删除所有远程分支（除main外）
REMOTE_BRANCHES=$(git branch -r | grep -v "main" | grep -v "HEAD" | sed 's/origin\///' | xargs)
if [ -n "$REMOTE_BRANCHES" ]; then
  for branch in $REMOTE_BRANCHES; do
    git push origin --delete "$branch" 2>/dev/null && echo "✅ 删除远程分支: $branch" || echo "⏭️  跳过: $branch"
  done
else
  echo "ℹ️  没有其他远程分支需要清理"
fi

# 清理远程分支的本地引用
git remote prune origin

echo "✅ 分支清理完成，只保留 main 分支"
```

## 配置要求

执行此skill前，确保：

1. **环境变量已设置**:
```bash
export GITHUB_PERSONAL_ACCESS_TOKEN="your_token"
```

2. **SSH密钥已配置**:
```bash
ssh -T git@github.com
```

3. **项目目录正确**:
在Git仓库根目录执行

遇到错误时：
1. 显示详细错误信息
2. 提供解决建议
3. 询问用户是继续还是中止

## 自定义选项

可以通过参数自定义行为：

```
/push-test-merge --no-test     # 跳过测试
/push-test-merge --auto-merge  # 自动合并，不询问
/push-test-merge --keep-branch # 保留源分支
```
